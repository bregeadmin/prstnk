# ЁPRST — Дизайн и архитектура журнала

**Дата:** 2026-05-30  
**Проект:** prstnk.ru — магазин авторской печатной графики  
**Статус:** утверждён к реализации

---

## Контекст

PRSTNK — магазин печатной графики петербургских художников. Журнал **ЁPRST** — контент-раздел сайта: статьи про офорт, линогравюру, художников, мастерские. Позиционирование: «мы не арт-институция, мы просто продаём и пилим контент про искусство».

Три уровня контента:
- **Ежедневно** — посты в Telegram (@prstnk_store) → секция «Лента»
- **Еженедельно** — отдельные статьи → секция «Свежие материалы»
- **Ежемесячно** — скомпилированный выпуск ЁPRST → секция «Выпуски месяца»

---

## Страницы к реализации

| URL | Описание |
|-----|----------|
| `/journal` | Главная журнала — текущий выпуск + лента + последние 3 выпуска |
| `/issues` | Архив всех выпусков |
| `/issues/04` | Страница конкретного выпуска (список статей) |
| `/journal/<slug>` | Страница отдельной статьи |

## Структура страницы `/journal`

### 1. Герой (Hero)
Grid 56fr / 44fr.
- **Левая колонка:** фото/иллюстрация главной статьи выпуска (full-bleed)
- **Правая колонка:** кикер (рубрика · тип), заголовок H1 с курсивным выделением через `<em>`, лид-абзац, кнопка «Читать →», штамп выпуска внизу
- На мобиле — стекируются вертикально

### 2. Рубричный навбар
Горизонтальная строка ссылок: Всё / Разговоры / На пальцах / История / Как смотреть / Лента. Каждая — ссылка на отдельную страницу рубрики (`/journal/razgovory`). Активная рубрика — инверсия (чёрный фон). Счётчик статей в `<sup>`. Без JS-фильтрации на клиенте (статик-сайт).

### 3. Свежие материалы (Editorial grid)
Grid 1.75fr / 1fr:
- **Лид-статья (левая):** изображение full-bleed, рубрика, H2, лид, мета (время чтения + дата)
- **Стек (правая):** 2 карточки друг над другом, разделены чертой. Каждая — рубрика + H3 + мета

### 4. Горизонтальный фичер
Full-width, grid 1fr / 1fr (без колонки с номером — убрана):
- **Левая:** рубрика, H2 с курсивом, лид, мета, кнопка «Читать →»
- **Правая:** pull-quote курсивом (border-left алый), краткое описание, автор

### 5. Лента
Blueprint-стиль (намеренно другой жанр — no images):
- **Левая колонка (220px):** заголовок «Лента. *Каждый день.*», подпись «из Telegram · между выпусками», ссылка @prstnk_store
- **Правая:** 4 карточки в grid 4×1. Каждая — дата (моно), тег (#репортаж), заголовок, «Читать в Telegram»
- Верхняя граница — хвоя (`--c-hvoya`)

### 6. Выпуски месяца
- Фон: серый `#E4E0DA`
- Шапка: «Выпуски *месяца*.» слева + «ВСЕ ВЫПУСКИ →» справа (ссылка на `/issues`)
- Grid 3×1 — последние 3 выпуска
- **Система цветов обложек:** каждый номер = свой цвет из палитры PRSTNK:
  - № 04 = алый `#FA2A22`
  - № 03 = кобальт `#2A4BFF`
  - № 02 = фуксия `#FF3DA0`
  - № 01 = хвоя `#2E5A2A`
  - (цикл по 5 цветам палитры для будущих выпусков)
- Каждая обложка: цветной фон, призрачный номер (opacity .1), ЁPRST, мета, имя/тема выпуска, подзаголовок

## Интеграция с build.py

Новые функции в `build.py`:
- `render_journal_index()` → `/journal.html`
- `render_issues_index()` → `/issues.html`
- `render_issue_page(issue)` → `/issues/<n>.html`
- `render_article(article)` → `/journal/<slug>.html`
- Вспомогательная `issue_color(n)` — возвращает цвет обложки по номеру выпуска (цикл по 5 цветам палитры)

Все функции читают JSON из `data/journal/`, генерируют HTML через шаблонные строки (как остальные функции build.py). Чистые ссылки без `.html` — через существующую `clean_links()`.

## CSS

Новый файл `journal.css`, подключаемый только на страницах журнала (в `<head>` через `build.py`). Это не раздует `prstnk.css` и позволит изолированно работать со стилями журнала. Все классы с префиксом `.jn-` чтобы не конфликтовать с существующими стилями сайта.

Существующие стили из `_journal-mockup.html` переносятся в файл с переименованием. Адаптивность: мобильный брейкпоинт 780px (стекирование колонок), 480px (1-колонка).

## Ограничения

- Без сервера: всё статично, нет JS-фильтрации по рубрикам на старте (можно добавить позже как progressive enhancement)
- Изображения: пока SVG-заглушки; реальные фото — через Sveltia CMS
- `_journal-mockup.html` — временный файл, удалить перед первым коммитом
